## 二项式系数(组合数)的计算
### 杨辉三角
`C(n, m) % p = (C(n - 1, m) + C(n - 1, m - 1)) % p`, 没什么好说的, 复杂度$O(n^2)$.
```cpp
mint _C[N][N];
constexpr int N = 2026;
void init_pascal() {
    static bool inited = 0;
    if (inited) {
        return;
    }
    for (int i = 0; i < N; i++) {
        _C[i][0] = _C[i][i] = 1;
        for (int j = 1; j < i; j++) {
            _C[i][j] = _C[i - 1][j] + _C[i - 1][j - 1];
        }
    }
    inited = 1;
}
mint nCr(ll n, ll m) {
    if (m < 0 || m > n) {
        return 0;
    }
    init_pascal();
    return _C[n][m];
}
```

### 线性预处理逆元
1. 求出前`n`个整数的阶乘和阶乘逆元`fact[n + 1], factinv[n + 1]`.
2. 使用定义`C(n, m) = fact[n] * factinv[m] * factinv[n - m]`即可.

模数应为大于`n`的素数, 否则无法保证逆元存在.

复杂度$O(n), O(1)$
```cpp
ll MOD = 998244353;
vector<ll> fact, factinv;
constexpr int N = 2e5 + 50;
void init_fact() {
    static bool inited = 0;
    if (inited) {
        return;
    }
    int n = N;
    inited = 1;
    ll p = MOD;
    fact.resize(n + 1);
    factinv.resize(n + 1);
    fact[0] = 1;
    for (int i = 1; i <= n; i++) {
        fact[i] = mul(fact[i - 1], i, p);
    }
    factinv[n] = inv(fact[n], p);
    for (int i = n - 1; i >= 0; i--) {
        factinv[i] = mul(factinv[i + 1], i + 1, p);
    }
    inited = 1;
}

ll nCr(ll n, ll m) {
    if (m < 0 || m > n) {
        return 0;
    }
    ll p = MOD;
    init_fact();
    ll ret = fact[n];
    ret = mul(ret, factinv[m], p);
    ret = mul(ret, factinv[n - m], p);
    return ret;
}    
```


### 直接计算
注意到`C(n, m) = _fact[n] / (_fact[m] * _factinv[n - m]) = (n * (n - 1) * ... * (n - m + 1)) / (m * (m - 1) * ... * 1)`(上面`m`项, 下面`m`项). 所以只需要$O(m + log P)$的复杂度即可直接计算. 在`n`很大, `m`很小的时候很好用.
```cpp
ll MOD = 998244353;
ll nCr(ll n, ll m, ll p = MOD) {
    if (m < 0 || m > n) {
        return 0;
    }
    if (m > n / 2) {
        m = n - m;
    }
    ll A = 1;
    ll B = 1;
    for (ll i = 0; i < m; i++) {
        A = mul(A, n - i, p);
        B = mul(B, m - i, p);
    }
    return mul(A, inv(B, p), p);
}
```

### Lucas定理
`C(n, m) % p = C(n / p, m / p) * C(n % p, m % p)`. 在`p`较小的时候有用.

```cpp
ll Lucas(ll n, ll m, ll p) {
    if (m == 0) {
        return 1;
    }
    return mul(Lucas(n / p, m / p, p), nCr(n % p, m % p, p), p); // 通常使用预处理版本的nCr, 注意这里的nCr要求在m > n时返回0, 别忘记了.
}
```

### 高精度求组合数
直接用`BigInt`乘超级慢. 这里介绍一种使用`Legendre`定理优化的方法.
1. 先筛n以下的质数.
2. 注意到`C(n, m) = fact[n] / (fact[m] * fact[n - m]`. 且`fact[n]`中一定包含且只包含所有小于等于`n`的质数作为质因数. 我们枚举小于等于`n`的质数`p`, 处理出`fact[n]`中`p`的幂次, 减去`fact[m]`中的幂次, 再减去`fact[n - m]`中的幂次, 即为`C(n, m)`中质数`p`的幂次.
3. 快速得到`fact[i]`的幂次可以使用**Legendre定理**, 即`fact[i]`中`p`的幂次为`i / p + i / p^2 + ... + i / p^k`. 这是个有限项的求和.
```cpp
vector<int> primes;
int N = 2e6 + 50;
void sieve(int n = N) {
    // 埃氏筛或线性筛
}
ll legendre(ll n, ll p) {
    // 返回 n! 中, p的幂次
    ll ret = 0;
    while (n > 0) {
        ret += n / p;
        n /= p;
    }
    return ret;
}
BigInt nCr(ll n, ll m) {
    if (m < 0 || m > n) {
        return 0;
    }
    BigInt ret = 1;
    for (int i = 0; primes[i] <= n; i++) {
        int p = primes[i];
        int exp = legendre(n, p) - legendre(m, p) - legendre(n - m, p);
        ret *= BigInt(p).pow(exp);
    }
    return ret;
}
```


## 概率和期望
本章涵盖概率dp和期望dp的内容.

### 期望的线性性
计算期望最有用的工具, 比如计算某集合$A$中满足某性质的子集$B$的元素个数的期望是多少. 可以设定指示器变量$I_i$(若$a[i]$在$B$中为1, 否则为0). 求$B$元素个数期望即转化为求$E[\sum_i I_i]$. 由于期望的线性型, 我们可以把$\sum_i I_i$拆开, 转化成求$\sum_iE[I_i]$, 设$p_i$为$a[i]$在$B$中的概率, $E[I_i] = 1 \cdot p_i + 0 \cdot (1 - p_i) = p_i$. 这时候对$p_i$求和即为所求结果.

例题: [牛客126319F](https://ac.nowcoder.com/acm/contest/126319/F)

简单来记的话就是"概率dp正推, 期望dp倒推". 这是因为算概率的时候通常需要计算从起点到某状态的可能性, 而算期望的时候通常计算从当前状态到终点的期望代价.

### 概率的转移(概率dp)
我们把状态看作点(每个状态拥有一个状态概率$p_x$), 状态的转移看作有向边, 终点为$y$, 每一条有向边赋予一个转移概率(流量)$p_{xy}$. 所有初始状态的概率和为1. 若存在终止状态, 则所有终止状态的概率和也为1. 每个节点的所有转移概率的和为1. 大致可以分成以下三类.

#### DAG上
假设当前在$x$点, 状态概率为$p_x$. 对于每条出边概率$p_{xy}$有$p_y += p_{xy} \times p_x$. 同理可以倒过来得到$p_x$等于所有入边概率乘以前驱的状态概率.

#### 带自环的DAG上
假设当前$x$点状态概率为$p_x$, 自环转移概率为$p'$, 对于除自环外的出边$y$, $p_{xy}$来说, 我们需要对所有概率进行归一化, 即所有出边的概率都要除以$1 - p'$, 这时候就消除了自环. (很好理解, 因为如果$p'$小于1, 那就不可能一直走自环, 总有一天会走到另外的出边上). 所以$p_y += \frac{p_xp_{xy}}{1 - p'}$. 注意到这种情况是不能倒过来用前驱来计算当前节点的, 因为每条入边归一化的分母$p'$都不一样(应为前驱节点的$p'$, 而非本节点的$p'$).

### 期望的转移(期望dp)

#### DAG上
先计算终点期望, 然后反推到起点.

对于每条出边`y, p`:

$$E[x] = \sum_{(x, y) \in E} p_{xy} \cdot (E[y] + w_{xy})$$

其中 $p_{xy}$ 是从 $x$ 转移到 $y$ 的概率, $w_{xy}$ 是这次转移的代价.

#### 带自环的DAG上
假设点 $x$ 有 $p'$ 的概率走自环(代价 $w'$).有 $p_{xi}$ 的概率走到 $y_i$(代价 $w_{xi}$):

$$E[x] = p'(E[x] + w') + \sum p_{xi}(E[y_i] + w_{xi})$$

移项化简可得: 
$$E[x] = \frac{p' w' + \sum p_{xi}(E[y_i] + w_{xi})}{1 - p'}$$
*注: 若每步代价均为 1, 则公式简化为 $E[x] = \frac{1 + \sum p_{xi} E[y_i]}{1 - p'}$.*
